#### 指针

**~~指针的定义与使用~~**

指针的作用：

- 可以通过指针间接访问内存
- 指针保存着变量所在的内存地址

---

定义与使用

```cpp
int main()
{
  int a = 10;

  // 1.  定义指针
  int *p;
  p = &a; //&a获取变量a所在的内存地址
  // 2. 使用指针 (de-reference:得到该内存中储存的数据)
  *p = 1000;
  cout << a << endl; // 1000
}
```

- `*P = 1000;` ❌

> 📣 **野指针**
>
> 指向 _非法内存空间_ 的指针称为野指针
>
> - 非法内存空间：没有申请的内存空间

**~~const 修饰指针~~**

const 修饰指针的三种情况 (关注 const 后面跟的是谁)

- const 修饰的是指针 - 常量指针
- const 修饰的是常量 - 指针常量
- const 既修饰指针，又修饰常量

```cpp
int main()
{
  int a = 10;
  int b = 10;
  // 普通指针
  int *p = &a;
  // 常量指针:指针所指向内存中存储的值要是不变的
  const int *p = &a;
  p = &b; ✔
  // 指针常量：指针的指向不可以改
  int * const p = &a;
  // 指针的指向 和 指针指向的变量值都不可以改
  const int * const p = &a;
}
```

**~~指针配合数组和函数案例~~**

案例描述

- 封装一个函数
- 利用冒泡排序，实现对整型数组的升序排序
- 数组名（如 `arr`）本质上是一个指向数组第一个元素的指针

```cpp
void bubbleSort(int* arr, int len)
{
  for (i = 0; i < len-1 ; i++)
  {
    for (j = 0; j < len-i-1; j++)
    {
      if (arr[j] > arr[j+1])
      {
        int temp = arr[j];
        arr[j] = arr[j+1];
        arr[j+1] = temp;
      }
    }
  }

}
int main()
{
  int arr[10] = {4,3,6,9,1,2,10,8,7,5};
  int len = sizeof(arr) / sizeof(arr[0]); // 数组总内存 / 一个元素占用的内存 = 长度
  // 数组名（如 arr）本质上是一个指向数组第一个元素的指针
  bubbleSort(arr, len);
}
```

#### 程序的内存模型

不同区域存放的数据有着不同的生命周期
程序编译后会生成 `.exe` 程序，而未执行该程序前会分为代码区和全局区两个区域

_代码区_ ：存放函数体的二进制代码，由操作系统进行管理

_全局区_：存放全局变量、静态变量和常量

_栈区（stack）_ ：由编译器自动分配和释放，存放函数的参数值、局部变量等

_堆区（heap）_ ：手动分配和释放

<br></br>
**~~内存四区 - 代码区~~**

代码区特点

- 存放 CPU 执行的机器指令
- 代码区是共享的
- 代码区是只读的

<br></br>
**~~内存四区 - 全局区~~**

全局区特点

- 存放全局变量和静态变量
- 全局区中有常量区，用于存放字符串常量和 const 修饰的全局常量
- 该区域的数据再程序结束后由操作系统释放

<br></br>
**~~内存四区 - 栈区~~**

栈区特点

- 由编译器自动分配释放
- 存放函数的参数值、局部变量等
- 不要返回局部变量的地址，因为该内存在离开作用域后会被自动释放

<br></br>
**~~内存四区 - 堆区~~**

堆区特点

- 手动分配和释放
- 用 new 在堆区开辟内存

<br></br>

#### 引用

**~~引用的基本语法~~**

`数据类型 &别名 = 原名`

```cpp

```

<br></br>
**~~引用的注意事项~~**

引用必须初始化 `int &b;` ❌

引用在 _初始化后不可改变_

<br></br>
**~~引用做函数参数~~**

_值传递_

_地址传递_ （形参修饰实参）

_引用传递_ （形参修饰实参 + 简化指针地址传递）

```cpp
// 值传递
void swap1(int a, int b)
{
    int temp = a;
    a = b;
    b = temp;
}
// 地址传递
void swap2 (int* p1, int* p2)
{
    int temp = *p1;
    *p1 = *p2;
    *p2 = temp;
}
// 引用传递
void swap3 (int& r1, int& r2)
{
    int temp = r1;
    r1 = r2;
    r2 = temp;
}
int main()
{
    int a = 10;
    int b = 20;
    swap1(a, b); // 值传递
    cout << a << b << endl; // a = 10, b = 20

    swap2(&a, &b); // 地址传递
    cout << a << b << endl; // a = 20, b =10
    swap3(a,b); //引用传递
    cout << a << b << endl; // a = 20, b =10
}
```

#### 函数高级

**~~函数的默认参数~~**

函数的形参可以有默认值

语法：`返回值类型 函数名 （参数 = 默认值）{}`

注意事项

- 在调用函数没有传入参数的情况下，使用默认值。有传入参数时，使用传入的值
- 如果某个位置已经有了默认参数，那么从这个位置往后，从左到右都必须有默认值
- 如果函数声明时（无 body）有默认参数，那么函数实现（有 body）就不能有默认参数

<br></br>
**~~函数的占位参数~~**

函数的形参列表中可以有占位参数，调用函数时必须填补该位置

语法：`返回值类型 函数名（数据类型）{}`

```cpp
void func(int a, int)
{
    ...
}
int main()
{
    func(10,10);
    func(10); //❌ 没有填补占位参数位置
}
```

<br></br>

**~~函数重载基本语法~~**

函数名可以重用，提高复用性

函数重载需要满足条件：

- 同一作用域下
- 函数名称相同
- 函数类型不同或个数不同或顺序不同

函数如果 _只有返回值不同_ ， _不满足_ 重载条件

```cpp
void func()
{
    ...
}
void func(int a)
{
    ...
}
int main()
{
    func();
    func(10);
}
```

<br></br>

**~~函数重载注意事项~~**

引用可以作为重载条件

```cpp
void func(int& a)
{
    cout << "func(int& a) was called" << endl;
}
void func(const int& a)
{
    cout << "func(const int& a) was called" << endl;
}
int main()
{
    //  "func(int& a) was called" 调用了第一个函数
    // a本身是变量，属于int&类型而不是const int&类型
    int a = 10;
    func(a);
    // "func(const int& a) was called" 调用了第二个函数
    // 因为20作为引用会自动生成const int& temp = 20. 然后temp作为一个常量引用参数调用第二个函数
    func(20);
}
```

---

函数重载碰到函数默认参数

```cpp
void func(int a)
{
    cout << "func(int a) was called" << endl;
}
void func(int a, int b = 10)
{
    cout << "func(int a, int b = 10) was called" << endl;
}
int main()
{
    func(20); //❌ 函数调用有歧义
}
```

#### 类和对象 - 对象的特征

**~~构造函数和析构函数~~**

对象的初始化（构造函数）与清理（析构函数）

这两个函数会被编译器 _自动调用_ ，完成对象初始化和清理工作

如果我们不提供构造和析构函数，编译器会提供编译器提供的构造函数和析构函数（空实现）

---

构造函数

- `类名(){}`
- 可以有参数，可以重载

---

析构函数

- `~类名(){}`
- 无参数，不可以重载

<br></br>
**~~函数的分类以及调用~~**

分类

- 按参数分为：有参构造和无参构造
- 按类型分为：普通构造和拷贝构造

---

调用

- 括号法
- 显示法
- 隐式转换法

---

```cpp
class Person
{
public:
    int age;
    // 普通构造函数
    Person(int age)
    {
        this.age = age;
    }
    // 拷贝构造函数 (用参数p的所有特征构造一个新的实例)
    Person(const Person& p)
    {
        age = p.age;
    }
};
int main()
{
    // 括号法调用
    Person p1(10);
    // 显示法调用
    Person p2 = Person(10); // Person p2(10)

    // 隐式转换法调用
    Person p4 = 10; // 相当于Person p4 = Person(10)
}
```

<br></br>
**~~拷贝构造函数调用时机~~**

使用拷贝构造函数的情况

- 使用一个 已经创建完毕的对象 来初始化一个新对象
- 值传递的方式给函数参数传值（值传递时会调用拷贝构造函数）
- 以值方式返回局部对象

---

```cpp
class Person
{
public:
    int age;
    // 普通构造函数
    Person(int age)
    {
        this.age = age;
    }
    // 拷贝构造函数 (用参数p的所有特征构造一个新的实例)
    Person(const Person& p)
    {
        age = p.age;
    }
    // 析构函数
    ~Person(){}
};
```

<br></br>
**~~构造函数调用规则~~**

默认情况下，c++编译器至少给一个类添加 3 个函数

- 默认构造函数
- 默认析构函数
- 默认拷贝构造函数（对属性进行值拷贝）

---

构造函数调用规则如下

- 如果用户有定义有参构造函数，那么 c++不提供默认无参构造函数，但会提供拷贝构造函数
- 如果用户有定义拷贝构造函数，c++不提供其他构造函数

#### 类和对象 - 多态

**~~多态的基本语法~~**

多态的分类

- 静态多态：函数重载、运算符重载都算静态多态，复用函数名
- 动态多态：子类和虚函数实现运行时多态

---

静态多态和动态多态的区别

- 静态多态：函数地址早绑定 - 编译阶段确定函数地址
- 动态多态：函数地址晚绑定 - 运行阶段确定函数地址

---

满足动态多态的条件

- 有继承关系
- 子类要重写父类的虚函数

```cpp
class Animal
{

};
class Cat : public Animal

void doSpeak(Animal& animal)

}
int main()
{
    Cat cat;

}
```

函数 doSpeak() 在编译阶段已经确定地址，地址早绑定

- 无论传入的是 Animal 或是 Animal 的子类，都会调用基类 Animal 的 speak() 函数

**~~多态的原理剖析~~**

当 _父类类型的指针_ 指向 _子类对象_ 时，发生多态

<br></br>
**~~纯虚函数和抽象类~~**

多态中，通常父类中的虚函数都会被子类重写，本身函数并无意义

纯虚函数 - 无函数体

- 语法 `virtual 返回值类型 函数名(参数列表) = 0;`

当一个类中存在纯虚函数，该类被称为抽象类

- 无法实例化抽象类对象
- 继承抽象类的子类必须重写抽象类中的纯虚函数，否则该子类也属于抽象类

---

```cpp
class Base
{
public:
    // 纯虚函数
    virtual void func() = 0;
};
class Son : public Base
{
public:
    // 重写父类纯虚函数
    void func()
    {
        ...
    };

};
int main()
{
    // 多态 - 父类的指针或引用指向子类对象
    Base* base = new Son;
    base->func(); //调用子类函数
}
```

<br></br>
**~~虚析构和纯虚析构~~**

使用多态时，如果子类中有属性开辟到堆区，父类的指针在释放时无法调用到子类的析构代码

- 即 `Base* base = new Son;delete base` 无法释放该内存，因为需要用到子类的析构函数

解决方法：讲父类的析构的函数改为 _虚析构_ 或 _纯虚析构_

---

虚析构和纯虚析构的共同特点：

- 解决父类指针释放子类对象
- 都需要有具体的函数实现

---

虚析构和纯虚析构的不同：

- 纯虚析构会让该类变为抽象类，无法实例化对象

---

虚析构语法：`virtual ~类名`

---

```cpp
class Animal
{
public:
    virtual void speak() = 0; //纯虚函数
    ～Animal(){}
};
class Cat : public Animal
{
public:
    void speak(){} //重写纯虚函数
    // 构造函数中创建了堆区变量
    Cat(string name)
    {
        m_Name = new string(name);
    }
    // 析构函数释放堆区内存
    ~Cat()
    {
        if (m_Name != NULL)
        {
            delete m_Name;
            m_Name = NULL;
        }
    }
    string* m_Name;
};
int main()
{
    Animal* animal = new Cat;
    animal->speak();
    delete animal; // Cat的析构函数不会被调用，m_Name变量没有从堆区释放
}
```

```cpp
class Animal
{
public:
    virtual void speak() = 0; //纯虚函数
    virtual ～Animal(){} //虚析构
};
// 此时在释放指向子类的父类指针时，会调用子类的析构函数，因为父类的析构函数在子类中被重写
```

```cpp
class Animal
{
public:
    virtual void speak() = 0; //纯虚函数
    virtual ～Animal() = 0 //纯虚析构
};
// 纯虚析构的实现
Animal::~Animal(){}
```

纯虚析构需要声明也需要实现，但在类外实现，不在子类中实现

**~~读文本文件~~**

读 文本文件步骤

- 包涵头文件 `#include <fstream>`
- 创建流对象 `ifstream ifs;`
- 打开文件并判断是否打开成功 `ifs.open("文件路径"，打开方式)`
- 读数据： 四种方式读去
- 关闭文件 `ifs.close()`

```cpp
#include <fstream>
#include <string>
int main()
{
    ifstream ifs;
    ifs.open("fileName", ios::in);
    if (!ifs.is_open())
    {
        cout << "Open file Failed" <<endl;
        return;
    }
    // 读数据的四种方式
    // 第一种
    char buffer1[1024] = {0}; //创建一个容量为1024个元素的字符数组
    while (ifs >> buffer1)
    {
        cout << buffer1 << endl
    }
    // 第二种
    char buffer2[1024] = {0}; //创建一个容量为1024个元素的字符数组
    while (ifs.getline(buffer2, sizeof(buffer2)))
    {
        cout << buffer2 <<endl;
    }
    // 第三种
    string buffer3;
    while (getline(ifs, buffer3))
    {
        cout << buffer3 << endl;
    }
    // 第四种 （少用）
    char c;
    while ( c = ifs.get() != EOF)
    {
        cout << c << endl;
    }
    ifs.close();
}
```

#### 模板

> 📣 **泛型编程**
>
> C++ 的范型编程（Generic Programming）是一种编程范式，通过编写与类型无关的代码，使得程序可以处理不同数据类型，而无需为每种类型单独编写代码。
>
> C++ 的范型编程主要通过 模板（Template）机制实现，提供了灵活性和代码重用性。

<br></br>
**~~函数模版的基本语法~~**

模板不可以直接使用，只是一个框架，且模板的通用性并不是万能的

模板机制：

- 函数模板
- 类模板

---

函数模板：

- 建立一个通用函数，其返回值和形参的数据类型可以不具体定制，用一个虚拟的类型来代替
- 语法 `template <typename T>`

```cpp
//交换两个x类型数据
template <typename T> //声明一个模板
void mySwap (T& a, T& b)
{
    T temp = a;
    a = b;
    b = temp;
}
int main()
{
    int a = 10;
    int b = 20;
    // 自动类型推导使用函数模板
    mySwap(a,b);
    // 显示指定类型使用函数模板
     mySwap<int>(a,b);
}
```

<br></br>
**~~函数模版的注意事项~~**

自动类型推导，必须要推导出一致的数据类型 T，才可以使用

模板必须要确定出 T 的数据类型才可以使用

```cpp
//交换两个x类型数据
template <typename T> //声明一个模板
void mySwap (T& a, T& b)
{
    T temp = a;
    a = b;
    b = temp;
}
int main()
{
    int a = 10;
    int b = 20;
    int c = 'c';
    mySwap(a,c); // ❌ 自动推导出的数据类型不一致
}
```

<br></br>
**~~普通函数与函数模版的区别~~**

普通函数调用时，可以发生自动类型转换（隐式类型转换）

函数模板如果以自动类型推导调用，不会发生隐式类型转换

函数模板如果以显示指定类型的方式调用，可以发生隐式类型转换

```cpp
int myAdd1 (int a, int b)
{
    return a + b;
}
template <template T>
T myAdd2 (T a, T b)
{
    return a + b;
}
int main()
{
    int a = 10;
    char c = 'c'; // a - 97, c - 99
    cout << myAdd1(a,c) << endl; // 10 + 99 = 109 隐式类型转换
    cout << myAdd2(a,c) << endl; //❌ 自动类型推导不发生隐式类型转换
    cout << myAdd2<int>(a,c) <<endl;  // 109 显示指定类型，隐式类型转换

}
```

<br></br>
**~~普通函数与函数模版的调用规则~~**

当普通函数和模板函数的函数名一样时，优先调用普通函数

可以通过空模板参数列表来强制调用函数模板

函数模板可重载

如果函数模板可以产生更好的匹配，优先调用函数模板

```cpp
void myPrint (int a, int b)
{
    cout << "普通函数调用" <<endl;
}
template <template T>
void myPrint (T a, T b)
{
    cout << "函数模板调用" <<endl;
}
int main()
{
    int a = 10;
    int b = 20;
    // 优先调用普通函数
    myPrint(a, b); //“普通函数调用”
    // 通过空模板列表来强制调用函数模板
    myPrint<>(a, b); //"函数模板调用"
}
```

类模板没有自动类型推导的使用方式
类模板在模板参数列表中可以有默认参数


**~~类模版中成员函数的创建时机~~**

普通类中的成员函数一开始就可以创建

类模板中的成员函数在调用时才创建

```cpp
class Person1
{
public:
    void showPerson1()
    {
        cout << "Person1" << endl;
    }
};
class Person2
{
public:
    void showPerson2()
    {
        cout << "Person2" << endl;
    }
};
template <class T>
class MyClass
{
public:
    T obj;
    //类模板中的成员函数
    void func1()
    {
        obj.showPerson1();
    }
    void func2()
    {
        obj.showPerson2();
    }
}
int main()
{
    // 可以成功编译，因为类模板中的成员函数在运行时创建
    // 但对象类型和函数中冲突时，运行时报错
    MyClass<Person> m;
    m.func1();
    m.func2(); //❌报错
}
```

**~~类模版对象做函数参数~~**

类模板实例化出的对象，作为参数传入到函数中

传入方式

- 指定传入的类型 - 直接显示对象的数据类型
- 参数模板化 - 将对象中的参数变为模板进行传递
- 整个类模板化 - 将这个而对象类型 模板化进行传递

```cpp
template<class T1, class T2>
class Person
{
public:
    T1 m_Name;
    T2 m_Age;
    Person (T1 name, T2 age)
    {
        this->m_Name = name;
        this->m_Age = age;
    }
    void showPerson()
    {
        cout << this->m_Name << this->m_Age <<endl;
    }
};
// 1. 指定传入类型
void printPerson1(Person<string, int>& person)
{
    person.showPerson();
}
// 2. 参数化模板
template<class T1, class T2>
void printPerson2(Person<T1, T2>& person)
{
    person.showPerson();
}
// 整个类模板化
template<class T>
void printPerson3(T& person)
{
    person.showPerson;
}
int main()
{
    Person<string,int> p("Andy", 18);
    printPerson1(p); // √
    printPerson2(p); // √
    printPerson3(p); // √
}
```

**~~类模版与继承~~**

当碰到类模板继承时，需要注意：

- 当子类继承的父类是一个类模板时，子类在声明时要指定出父类中 T 的类型
- 如果不指定，编译器无法给子类分配内存
- 如果想灵活制定出父类中 T 的类型，子类也需要变为类模板

```cpp
template<class T>
class Base
{
    T m;
};
// 1. 通过在声明时指定父类中T的类型，来继承父类 (e.g <int>)
class Son1 : public Base<int>
{

};
// 2. 通过将子类变成类模板，来继承父类
template<class T1, class T2>
class Son2 : public Base<T1>
{
    T2 obj;
};
int main()
{
    Son2<int, char> son;
}
```

**~~类模版与友元~~**

类模板配合友元函数的类内和类外实现

全局函数类内实现 - 直接在类内声明友元即可

全局函数类外实现 - 需要提前让编译器知道全局函数的存在

```cpp
// 提前让编译器知道模板类Person的存在
template<class T1, class T2>
class Person;
// 全局函数类外实现
// 提前让编译器知道全局函数printPerson2()的存在
template<class T1, class T2>
void printPerson2<>(Person<T1, T2> p)
{
    cout << p.m_Name << p.m_Age << nedl;
}
template<class T1, class T2>
class Person
{

    friend void printPerson1(Person<T1, T2> p)
    {
        cout << p.m_Name << p.m_Age << nedl;
    }
    //全局函数类外实现
    // 添加空模板的参数列表，表示这是一个函数模板
    friend void printPerson2<>(Person<T1, T2> p)；
public:
    Person(T1 name, T2 age)
    {
        this->m_Name = name;
        this->m_Age = age;
    }
private:
    T1 m_Name;
    T2 m_Age;

{
```

#### STL

**~~STL 的基本概念~~**

STL: Standard Template Library

STL 从广义上 分为：容器(container)、算法(algorithmn)、迭代器(iterator)

- _容器_ 和 _算法_ 之间通过 _迭代器_ 进行无缝链接
- STL 几乎所有代码都采用了 _类模板_ 或 _函数模板_

STL 的六大组件:

<img
	src='/assets/CS/cpp/img7.png'
	alt='img7'
	style='width: 100%; height: auto; display:block; margin: 0 auto;'
/>

---

容器

- 容器就是将运用最广泛的数据结构实现出来
- 如数组、链表、树、栈、队列、集合、映射

---

序列式容器和关联性容器

- 序列式容器：强调值的存放顺序不变，序列式容器中的每个元素均有固定的位置
- 关联式容器：二叉树结构，排序，各个元素之间没有严格物理上的顺序关系

---

算法

- 质变算法和非质变算法
  - 质变算法：运算过程中会更改元素的内容（如拷贝、替换、删除）
  - 非质变算法：运算过程中不更改元素内容（如查找、计数、遍历等）

---

迭代器

- 提供一种方法来依序访问某个容器中的各个元素，而又无需暴露容器的内部表示方式
- 每个容器都有专属的迭代器
- 算法要通过迭代器才能访问容器中的元素

```cpp
#include <vector>

    {
        cout << *itBegin << endl;
        itBegin++;
    }
}
```

**~~函数对象（仿函数）~~**

函数对象的概念

- 重载了 函数调用 操作符的类，该类实例化出来的的对象称为函数对象
- 函数对象使用重载的（）时，行为类似于函数调用，所以称为仿函数
- 函数对象是一个类的实例，不是一个函数

---

特点

- 函数对象在使用时，可以有参数有返回值
- 函数对象可以作为参数传递

```cpp
class MyAdd
{
public:
    // 重载()操作符
    // 属于类内的成员函数
    int operator()(int v1, int v2)
    {
        return v1 + v2;
    }
    int count; //类自己的状态
};
int main()
{
    MyAdd myAdd; //创建函数对象
    myAdd(10,10); //函数对象在使用(), 看起来很像在调用一个函数
}
```
